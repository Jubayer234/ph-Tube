Question (1) : Discuss the scope of var, let, and const?

Answer : These are namely, 'let,' 'var,' and 'const' keywords. Since ES6 standard, the three keywords are available for users to declare variables without initializing them. All three of these keywords have similarities in their syntax for declaring variables. However, they differ in their usage and scope, as detailed in the below.

  • (1) Var : (i) Variables declared with var are in the function scope.
    (ii) It gets hoisted to the top of its scope and initialized undefined.
    (iii) Can be declared without initialization.
    (iv) Can be accessed without initialization as its default value is “undefined”
  • (2) Let : (i) Variables declared as let are in the block scope.
    (ii) It also got hoisted to the top of its scope but didn't initialize.
    (iii) Can be declared without initialization.
    (iv) Cannot be accessed without initialization, leading to 'referenceError'.
  • (3) Const : (i) Variables declared as let are in the block scope.
    (ii) It also got hoisted to the top of its scope but didn't initialize.
    (iii) Cannot be declared without initialization.
    (iv) Cannot be accessed without initialization, as it cannot be declared without initialization in the first place.

Question (2) : Tell us the use cases of null and undefined :

Answer : Undefined means a variable has been declared but has yet not been assigned a value.
Null is an assignment value. It can be assigned to a variable as a representation of no value.

Question (3) : What do you mean by REST API?

Answer : A REST API is a way of easily accessing web services without having excess processing. Whenever a RESTful API is called, the server will transfer to the client a representation of the state of the requested resource.
